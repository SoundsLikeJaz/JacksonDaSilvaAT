Loja de Eletrônicos

Um projeto simples criado no final da minha disciplina de Desenvolvimento de Serviços Web e Testes com Java onde colocamos em prática nossas hábilidades com consumo de APIs utilizando Spark e, pela primeira vez com Java, fizemos integração com um front-end.
O projeto em questão representa uma loja de eletrônicos que pratica revenda de celulares, computadores e fones de ouvido além de manter um cadastro de seus clientes no sistema (todos podem ser acessados pela navbar acima). Além de cadastrar o cliente, também há uma comunicação com a API Viacep para preencher o endereço do cliente automaticamente após a inclusão de um CEP e há uma relação de um-para-muitos entre cliente e produto, que representa o carrinho de compras do cliente.


Apresentação

Sou Jackson da Silva, um jovem estudante de Engenharia de Software no instituto INFNET atualmente finalizando meu segundo semestre, sendo este o bloco de desenvolvimento back-end. Destre as disciplinas deste bloco está a disciplina de Desenvolvimento de Serviços Web e Testes com Java, que trouxe a mim e outros colegas a oportunidade de aprender sobre a linguagem Java com o professor Elberth Moraes que, além de ministrar a disciplina, atua ativamente no mercado dentro da DataPrev.
Dentre os desafios que Elberth nos proporcionou dentro de nossos meses juntos, há não apenas esse AT, projeto ao qual você está vendo agora e disponibilizo aqui o código dele no GutHub, mas também tivemos os nossos TPs, que são trabalhos pontuais ao longo da disciplina.
Devo ressaltar que reconheço a simplicidade do projeto do meu AT, infelizmente não pude fazer algo tão complexo quanto eu gostaria devido a fatores como pouco contato com Spark. Por isso convido-o a visitar o trabalho criado ao longo dos meus TPs, o Starify .
Neste trabalho, que já possui um nível maior de complexidade, pude dedicar mais tempo e estudei o consumo da API do Spotify utilizando Spring e Gradle. Criei uma relação de um-para-muitos entre as classes Álbum, Faixa e Artista para adicionar dados do próprio Spotify dentre de minha aplicação além dos dados locais instanciados por mim mesmo. Apesar de ele, até o presente momento, ainda não possuir um front-end, convido você à visitá-lo também para ter uma visão mais vasta da minha evolução enquanto programador.
